# CareerSignal

CareerSignal is a Python portfolio project that monitors job postings from target companies, stores results in SQLite, scores jobs against target criteria, exports clean Excel reports, sends daily email summaries, and powers a Power BI dashboard.

The project is designed to show practical automation, data handling, SQL storage, reporting, dashboarding, and business-focused problem solving in one end-to-end workflow.

## Project Goal

CareerSignal helps job seekers monitor companies they care about instead of manually checking each career page every day.

The pipeline collects job postings from supported applicant tracking systems, normalizes the data into one consistent format, stores jobs in a SQLite database, detects newly discovered jobs, scores each job for relevance, exports results to Excel, sends daily email reports, and visualizes results in Power BI.

The project is built around a real job-search problem: keeping track of many company career pages without relying on inconsistent job alert emails.

## Filtering Strategy

CareerSignal is designed to support multiple job-search lanes.

The project does not only filter for accounting and finance roles. The filtering strategy also covers analyst, operations, compliance, data/reporting, supervisor, and public-utility-adjacent roles.

Current target lanes include:

* Accounting roles
* Finance roles
* General analyst roles
* Business analyst roles
* Operations analyst roles
* Compliance analyst roles
* Data/reporting analyst roles
* Plant supervisor roles
* Operations supervisor roles
* Water/wastewater or public-utility-adjacent roles
* Other realistic roles that fit the userвЂ™s background

The filtering strategy defines:

* Strong title matches
* Maybe title matches
* Weak or stretch title matches
* Excluded or heavily penalized titles
* Strong target locations
* Acceptable locations
* Remote and hybrid rules
* Seniority rules
* Experience requirement rules
* Sector-specific rules

This filtering strategy will feed into later match scoring refinement.

## Setup Instructions

### 1. Create and activate a virtual environment

```bash
python -m venv .venv
```

Windows Git Bash:

```bash
source .venv/Scripts/activate
```

Windows Command Prompt:

```cmd
.venv\Scripts\activate
```

Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment variables

Copy `.env.example` to `.env` and fill in the required values.

```bash
cp .env.example .env
```

Do not commit `.env` to GitHub.

## Official Normalized Job Shape

Every collector should return job dictionaries using the same structure:

```python
{
    "company_name": str,
    "source_ats": str,
    "external_job_id": str,
    "title": str,
    "location": str,
    "department": str,
    "job_url": str,
    "posted_date": str,
    "date_collected": str,
}
```

This keeps Greenhouse, Workday, and future ATS connectors compatible with the same database, scoring, email, Excel, and Power BI workflow.
